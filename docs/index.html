Your history never leaves your machine

Everything runs on localhost against a local SQLite database. Private & LAN hosts (localhost, 192.168.x, *.local) are never sent to liveness checks or AI providers, and you can mark your own hosts private or hide them entirely. AI is optional and pluggable — without a key, every other feature still works.

From a flat log to something you can think with

Five ways it interprets your history — not just lists it.

🔍

Search & browse

Full-text search (SQLite FTS5) and optional semantic search over every URL, grouped by true domain with a nested path tree.

🐇

Research sessions

Reconstructs your rabbit holes — bursts of browsing with no long pause — as ordered page trails, scored by how deep they went.

💡

Insights

On This Day, forgotten gems, your browsing rhythm, recurring routines, stalled threads to pick back up, and a dead-link graveyard.

🗺️

Interest map

Clusters your history into named topics by meaning (k-means over embeddings), with rising/declining trend indicators.

💓

Liveness checks

Finds which links are still alive vs dead / blocked / rate-limited, with a Wayback Machine fallback for the dead ones.

🌐

Many sources

Import from Google Takeout or directly from Chrome, Edge, Brave, Vivaldi, Opera, Firefox & Safari — they merge and dedupe.

Up and running in a minute

One dependency: Bun ≥ 1.3. No native build step.

# clone & install
git clone https://github.com/joshuatownsend/chrome-history-explorer
cd chrome-history-explorer
bun install

# load history (Takeout export, or a local browser profile)
bun run ingest                 # reads ./History.json
bun run ingest --list          # or detect installed browsers

# build the UI and serve everything at http://127.0.0.1:8787
bun run build
bun run server

Loopback-only by default. See the README for AI setup, importing, and exposing it safely on a LAN.
